Security

Feature: Security is a feature, no less important than design or performance. Strong security must be built in from the ground up, not as an afterthought.

Design

Calm Technology: Information overload is an epidemic. Calm Technology is the cure.

No UI: Albert Einstein once said, "Everything should be made as simple as possible, but no simpler." Were he a web developer, that might instead have been "the best interface is no interface.

Explorable: The most intuitive interfaces are the most explorable ones. Content is reactive, feedback is immediate, there is no penalty for guessing, the way back is clear, entered data is preserved, yet nothing is permanent until explicitly made so.

Mobile First: Smaller screen sizes force designers to focus the UI and content, which adds value no matter what the screen size. Mobile First design is universal design.

Performance: Performance is a feature, in all respects: application response time, workflow efficiency, cognitive load, and UI/UX intuitiveness. Lag breaks flow.

Strategy

Fundamentals: How can the web help you increase revenue, decrease costs, or avoid losses?

Identify the Problem:: The stated problem is not always the root problem. Correctly identify the deeper, fundamental, root problem, and the solution often presents itself.

Increase Revenue: How can your web presence help you win more mindshare, and convert a greater proportion of potential customers to actual customers?

Reduce Costs: How can your web presence help you retain customers, provide ongoing value, and reduce churn and turnover?

Avoid Loss: How can your web presence help you minimize risks, especially of catastropic loss? What must you do with your web presence to protect it from such?

Data: How can we use data-driven due dilligence to answer these questions?
