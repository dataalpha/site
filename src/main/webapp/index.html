values

Security

Integral: Security is a feature, no less important than design or performance. Strong security must be built in from the ground up, not added as an afterthought.

Prevention: FOSS and the Cloud enable even organizations below the security poverty line to benefit from the work and knowledge of world class security professionsals. Stand on the shoulders of giants. Leverage services and software for which security is a top priority.

Framework: Transparent, mature, battle-hardened web frameworks built from scratch for strong security.

Hosting: World class security expertise as a service.

Testing: The more we sweat in practice, the less we bleed in battle. Prelaunch pentesting, third party auditing as justified by loss expectations [PDF].

Information: Keep abrest of the state of Internet security. Know your enemy and know yourself.

Design

Zen: Information overload is an epidemic. Calm technology is the cure.

Minimal: Albert Einstein once said, "Everything should be made as simple as possible, but no simpler." Were he a web developer, that might instead have been "the best interface is no interface.

Focused: Smaller screen sizes force designers to focus the UI and content, which adds value no matter what the medium. Mobile First design is universal design.

Explorable: The most intuitive interfaces are the most explorable ones. Content is reactive, feedback is immediate, there is no penalty for guessing, the way back is clear, entered data is preserved, yet nothing is permanent until explicitly made so.

Performance: Performance is a feature composed of response time, workflow efficiency, cognitive load, and UI/UX intuitiveness. Lag breaks flow.

Strategy

Fundamentals: How can the web help you increase revenue, decrease costs, or avoid losses?

Problems: "Corollary the First: ‘That every problem contains its own solution, and the task lies in the accurate statement of the nature of the problem.’" - Louis Sullivan, mentor of Frank Lloyd Wright

Revenue: How can your web presence help you win more customers, and remove barriers to customer development?

Costs: How can your web presence help you reduce costs, do more with less, reduce customer service demands, and enable customer self-service?

Loss: How can your web presence help you retain customers, reduce churn and turnover, and minimize risks, especially of catastropic loss?

Data: How can we use data-driven due dilligence to answer these questions?
